Privacy Policy

Effective date: 2025-08-14 · Version 1.0

SummarizeAI (the “App”) provides text summarization, OCR, PDF export, and related features. We respect your privacy and are committed to transparently explaining what data we collect, how we use it, and the choices you have.

What We Collect

User‑Provided Content

  • Text you enter for summarization
  • Images you select or capture for OCR/summarization
  • Audio you record for transcription (if used)

Usage & Interaction Data

  • Feature usage events (e.g., export, view, search, filters)
  • Summary type selections
  • Timestamps (stored in JST / Asia/Tokyo)

Device & Diagnostics

  • Pseudonymous user ID (derived from device ID)
  • Device model & OS version
  • Approximate IP (for security & logging)

Where Your Data Lives

  • On‑device: OCR via Apple Vision runs on your device. PDFs are generated locally. Saved summaries are stored locally on-device.
  • DeepSeek AI (cloud): When you request cloud summarization, the App sends the relevant text and, for image summaries, a resized/compressed image to DeepSeek’s API to generate the result.
  • Supabase (cloud): We store usage logs and interaction records in our Supabase project.

Details about Interaction Records

For transparency, our user_interactions table may store the following fields:

  • id (UUID), user_id (pseudonymous), device_id
  • interaction_type (e.g., text_summarization, image_summarization)
  • interaction_source (e.g., text, image, saved_summary)
  • raw_data: The original text you provided or OCR extracted text (full text)
  • processed_data: The generated summary content (full text)
  • result_summary: Short form of the result (may be same or omitted)
  • metadata: JSON with context (e.g., summary type, image dimensions)
  • created_at, updated_at: Timestamps stored in JST

Access logs (e.g., app launch, authentication events) may record device model, OS version, pseudonymous user ID, IP address, and location metadata (approximate).

How We Use Data

  • Provide core features (summarization, OCR, export)
  • Improve reliability and UX (troubleshooting, performance)
  • Security and abuse prevention
  • Analytics about high‑level usage patterns (non‑marketing)

Consent & Controls

  • You can use Guest Mode (no account) or authenticate. A pseudonymous ID is used in both cases.
  • Access logging and interaction logging can be managed via the in‑app consent prompt and Settings.
  • You can delete individual summaries in the App, and you can request deletion of interaction records (see “Your Rights”).

Third‑Party Services

  • Apple Vision: On‑device OCR. Data stays on your device for OCR.
  • DeepSeek AI: Cloud summarization service. We send only the text you request to summarize and, for image summaries, the resized/compressed image. Responses are used to generate your summary.
  • Supabase: Managed backend for logs and interaction data (see above fields).

Each provider processes data under its own terms & privacy policies.

Legal Basis (where applicable)

  • Consent for sending content to DeepSeek AI for cloud summarization and for storing interaction data.
  • Legitimate interests for security, fraud prevention, and service reliability.

Retention

  • On‑device summaries remain until you delete them.
  • Cloud interaction records are retained as needed for functionality and troubleshooting and deleted upon your request, subject to legal obligations.

Security

We use industry‑standard measures to protect data in transit and at rest. No method is 100% secure.

Children’s Privacy

The App is not directed to children under 13. If you believe a child has provided us data, contact us to delete it.

International Transfers

Servers used by our providers may be located in various regions. Timestamps are stored in JST (Asia/Tokyo) for consistency.

Your Rights

  • Access a copy of your interaction data
  • Request correction or deletion
  • Withdraw consent for logging or cloud summarization

To exercise rights, use in‑app controls or contact us.
